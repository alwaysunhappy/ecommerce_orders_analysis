# Учебный аналитический проект: анализ клиентского опыта в e-commerce

## 1. Цель проекта

Цель проекта — провести исследовательский анализ данных интернет-магазина и определить факторы, которые связаны с низкой оценкой заказа покупателем.

В рамках проекта рассматривались следующие факторы:

- задержка доставки;
- категория товара;
- регион покупателя;
- стоимость доставки относительно цены товара;
- регион продавца.

## 2. Данные

В проекте использован публичный датасет Olist Brazilian E-Commerce Public Dataset.

Данные содержат информацию о заказах, покупателях, продавцах, товарах, оплатах, доставке и отзывах.

Основные таблицы:

- `orders` — информация о заказе и датах доставки;
- `order_items` — товары внутри заказа;
- `order_payments` — оплаты;
- `order_reviews` — отзывы покупателей;
- `customers` — покупатели и их регионы;
- `sellers` — продавцы и их регионы;
- `products` — товары и категории;
- `product_category_translation` — перевод категорий на английский язык.

## 3. Подготовка данных

На первом этапе CSV-файлы были загружены в SQLite-базу. Затем с помощью SQL была собрана аналитическая витрина `orders_mart` на уровне одного заказа.

В витрину были добавлены следующие признаки:

- стоимость товаров в заказе;
- стоимость доставки;
- доля доставки в стоимости заказа;
- средняя оценка заказа;
- флаг плохого отзыва;
- время доставки;
- задержка относительно ожидаемой даты доставки;
- категория товара;
- регион покупателя;
- регион продавца.

## 4. Метрики

В проекте были рассчитаны следующие метрики:

- количество заказов;
- GMV;
- средний чек;
- средняя оценка заказа;
- доля плохих отзывов;
- доля задержанных доставок;
- среднее время доставки;
- доля отменённых заказов;
- доля повторных покупателей.

Плохим отзывом считалась оценка `review_score <= 2`.

## 6. Визуализации

В проекте построены графики:

- распределение оценок заказов;
- средняя оценка для заказов с задержкой и без задержки;
- доля плохих отзывов для заказов с задержкой и без задержки;
- категории с высокой долей плохих отзывов;
- регионы с высокой долей задержек;
- среднее время доставки по регионам;
- связь доли доставки в стоимости заказа со средней оценкой;
- количество заказов и GMV по месяцам.

## 7. Основные выводы

Заполнить после запуска пайплайна на реальных данных.

Пример формата:

1. Заказы с задержкой доставки имеют более низкую среднюю оценку и более высокую долю плохих отзывов.
2. Несколько категорий товаров дают непропорционально высокий вклад в плохой клиентский опыт.
3. В ряде регионов покупателей наблюдается повышенная доля задержанных доставок.
4. Высокая доля стоимости доставки в цене заказа может быть связана с более низкой оценкой.

## 8. Рекомендации

Пример рекомендаций:

1. Для регионов с высокой долей задержек пересмотреть ожидаемые сроки доставки или логистические процессы.
2. Для категорий с высокой долей плохих отзывов провести отдельный анализ причин: доставка, качество товара, продавцы.
3. Ввести мониторинг продавцов по SLA: доля задержек, доля плохих отзывов, средняя оценка.
4. Проверить A/B-тест с более реалистичным отображением срока доставки для проблемных регионов.

## 9. Идея A/B-теста

Гипотеза: если показывать покупателю более реалистичный срок доставки для направлений с высокой исторической долей задержек, то доля плохих отзывов снизится.

Основная метрика:

- доля плохих отзывов `review_score <= 2`.

Дополнительные метрики:

- конверсия в заказ;
- GMV;
- cancellation rate;
- средняя оценка заказа;
- доля обращений в поддержку, если такая метрика доступна.

Guardrail-метрики:

- падение количества заказов;
- падение GMV;
- рост отмен.
